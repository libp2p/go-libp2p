# Mixnet Security Testing Plan with PentAGI

This document outlines the strategy for testing the security and anonymity properties of the `go-libp2p-mixnet-impl` using the PentAGI autonomous pentesting framework.

## 1. Test Environment
- **Network**: Docker network `pentagi-network` (172.20.0.0/24)
- **Nodes**:
  - `source` (172.20.0.10): Originates mixnet traffic.
  - `dest` (172.20.0.17): Receives and reconstructs traffic.
  - `relay1-6` (172.20.0.11-16): 6 relay nodes forming 2 parallel circuits (3 hops each).
- **Tooling**: PentAGI (AI-driven pentesting)

## 2. Targeted Attack Scenarios

### A. Information Leakage (Anonymity Check)
- **Objective**: Verify that relays do not have access to metadata beyond their immediate neighbors.
- **PentAGI Action**:
  - Scan relay nodes for diagnostic/status endpoints.
  - Attempt to intercept `HandleStream` traffic.
  - Check for "Onion Leakage": Can a relay see the final destination PeerID or the original source IP?

### B. Traffic Correlation
- **Objective**: Determine if an attacker controlling multiple relays can correlate shards via timing.
- **PentAGI Action**:
  - Monitor packet timing across `relay1` (entry) and `relay3` (exit).
  - Use AI reasoning to correlate packet sizes and intervals.
  - Report on the effectiveness of the current sharding/padding strategy.

### C. Protocol Robustness (Fuzzing)
- **Objective**: Ensure the `/lib-mix/relay/1.0.0` protocol is resilient to malformed inputs.
- **PentAGI Action**:
  - Send packets with invalid `CircuitID`.
  - Send oversized payloads to test `MaxPayloadSize` enforcement.
  - Send "circular" routes (onion layers that point back to the same node).

### D. Resource Exhaustion (DoS)
- **Objective**: Test the `maxBandwidth` and `maxCircuits` enforcement in `relay.Handler`.
- **PentAGI Action**:
  - Attempt to open 1000+ circuits to a single relay.
  - Saturate relay bandwidth and observe if it affects other circuits (Isolation test).

## 3. How to Run

1. **Start PentAGI**:
   ```bash
   cd pentagi
   docker compose up -d
   ```
2. **Deploy Mixnet Testbed**:
   ```bash
   # From root
   docker compose -f mixnet-test/docker-compose.yml up -d
   ```
3. **Trigger PentAGI Flow**:
   - Go to `https://localhost:8443`.
   - Start a flow with prompt: *"Test the security of the mixnet relay network at 172.20.0.11 through 172.20.0.16. Focus on anonymity leaks and protocol fuzzing."*

## 4. Expected Deliverables
- **Vulnerability Report**: Generated by PentAGI, detailing any discovered leaks or crashes.
- **Anonymity Score**: Analysis of how difficult it was for the AI to correlate traffic.
- **Stability Analysis**: Feedback on protocol resilience against malformed packets.
